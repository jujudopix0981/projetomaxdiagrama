# Projeto: Mapeamento de Processo de Cobrança

Este repositório foi criado para organizar e apresentar a entrega do estudo de caso sobre o processo de cobrança de uma empresa. 

O objetivo do trabalho foi mapear o funcionamento atual do processo (As-Is) e propor soluções estruturais para melhorar o fluxo (To-Be). Os diagramas foram modelados em código utilizando a linguagem PlantUML.

## 1. Mapeamento do Processo Atual (As-Is)
Nesta etapa, documentei o fluxo original descrito na proposta do exercício. É possível observar que o processo atual possui etapas bastante manuais, dependendo fortemente do envio de cartas físicas e do contato ativo por telefone realizado pelo call center.


## 2. Proposta de Melhoria (Processo To-Be)
Nesta segunda etapa, busquei otimizar o fluxo para torná-lo mais ágil e reduzir os custos operacionais da empresa. As principais melhorias propostas incluem:

* Priorização de canais digitais (como e-mail e mensagens) antes da emissão de correspondência física.
* Inclusão de um portal de autoatendimento, dando autonomia para o devedor negociar a dívida sem a necessidade de um atendente.
* Implementação de uma análise de crédito (score) antes do acionamento de um cobrador externo, direcionando esse recurso mais caro apenas para casos com real chance de recuperação.
